Тестовое задание: отчет по доставкам

В рамках тестового задания тебе предстоит выступить в роли фулстек-разработчика и реализовать мобильное и веб-приложение по доставке. Мы хотим оценить не только технический уровень, но и твою самостоятельность, умение решать задачи в условиях неопределённости и понимать, как структурировать проект с нуля.
Мы активно используем AI-инструменты (включая Cursor AI), чтобы ускорять решение типовых задач, и пишем код руками там, где это быстрее и понятнее. Поэтому нам важно увидеть, как ты сочетаешь ИИ и инженерный подход.

Задача

Реализовать мобильное и веб-приложение для создания и учета доставок.
Минимально — работающее веб-приложение.
Максимально — мобильное приложение в формате APK.
В основе — тёмная тема и стилистика Material 3.
Реализовать проект необходимо с использованием Cursor AI.
Прототип приложения в Фигме
Архив с макетами экранов на Яндекс Диске
Макет веб-приложения
*создано ИИ, могут требоваться корректировки

Технологии

Frontend:
React Native для мобильной версии
React (Material 3, тёмная тема) для веб-отчёта
Backend: Django + Django Admin
Справочники
Авторизация
Обработка данных доставок
БД: PostgreSQL
Прочее:
Поддержка офлайн-режима (по желанию)
Рекомендуется использовать Cursor AI для реализации

Авторизация

Вход в приложение по логину и паролю
Регистрация не требуется — пользователи создаются вручную через Django Admin

Справочники (реализовать через Django Admin)

Модели транспорта (номер — произвольный текст при создании доставки)
Типы упаковки (например: "Пакет до 1 кг", "Целофан", "Коробка")
Услуги (например: "До клиента", "Хрупкий груз", "Мед.товары", и т.п.)
Статусы доставки (например: "В ожидании", "Проведено")
Типы груза (опционально, по желанию)

Функциональность мобильного приложения

Создание новой доставки:
Выбор модели транспорта (из справочника)
Ввод времени отправки и доставки
Указание дистанции в км (адреса — опционально)
Загрузка медиафайла (PDF, фото и т.д.)
Выбор услуги, упаковки, статуса, технического состояния
Просмотр списка доставок:
В карточке доставки — время, дистанция, услуга, упаковка, статус
Возможность редактировать, удалить, провести доставку
Стилизация: тёмная тема, ориентир на предоставленные макеты (Material 3)

Примечания

Используй чистый, читаемый код с комментариями
Обработка ошибок обязательна
Offline-режим и интеграция карт (например, Яндекс) — опциональны, но будут плюсом
Все визуальные компоненты можно адаптировать под Material 3 (макеты — ориентир, не догма)

 Описание экранов приложения

1. Экран «Создание новой доставки»

Название экрана: Новая доставка
Цель: Ввод данных о новой доставке
Поля и действия:
Модель и номер — выбор из справочника (модель), ручной ввод номера (например: V01, №123)
Время в пути — дата и время отправки и доставки (с расчетом продолжительности)
Дистанция — ввод расстояния вручную (в км), либо через карту (опционально)
Медиафайл — загрузка одного файла с устройства (PDF, изображение)
Услуга — выбор одной или нескольких опций из справочника
Статус доставки — выбор из справочника
Упаковка — выбор одной опции из справочника
Техническое состояние — выбор из двух состояний: Исправно, Неисправно
Кнопка: «Создать»

2. Экран «Редактирование доставки»

Название экрана: №[Номер доставки]
Цель: Просмотр и редактирование информации по доставке
Функциональность:
Все поля аналогичны экрану «Создание»
Кнопки:
Провести — переводит статус в «Проведено»
Удалить — удаляет доставку

3. Экран «Список доставок»

Название экрана: Доставка
Цель: Просмотр всех доставок
Элементы:
Список карточек доставок: номер, дистанция, время, услуга, упаковка, статус, тех. состояние
Статусы отображаются цветными метками:
Проведено — зелёный
В ожидании — жёлтый
Кнопка "+" для создания новой доставки
Фильтры:
По времени в пути
По дистанции

4. Экран «Адреса и координаты»

Цель: Ввод координат вручную или выбор на карте (опционально)
Поля:
Откуда
Куда
Дистанция (в км)
Кнопка: «Применить»

5. Экран «Упаковка»

Цель: Выбор упаковки из справочника
Примеры значений:
Пакет до 1 кг
Целофан
Коробка
Бумажный пакет
Пластиковый контейнер
Ящик
Нет упаковки

6. Экран «Услуга»

Цель: Выбор одной или нескольких услуг
Примеры значений:
До клиента
Перемещение между складами
Физ. лицо
Юр. лицо
Мед. товары
Хрупкий груз
Температурный режим

7. Экран «Дата и время»

Цель: Ввод времени отправки, доставки и автоматический расчет времени в пути
Поля:
Дата отправки + время
Дата доставки + время
Время в пути (рассчитывается автоматически или указывается вручную)

8. Экран «Модель и номер»

Цель: Выбор модели из справочника и ручной ввод номера
Примеры моделей:
V01
X20
REX
Спринтер
Газель
Лада Груз
ЭлектроМобиль
